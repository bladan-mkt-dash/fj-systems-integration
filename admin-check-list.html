Suggested drill-downs

These turn the Admin Check List from a static daily PDF into an actionable ops console.

  1. By gap type — Click “Missing Credit Card” (or any field in the 5 Journeys Practice summary) → filtered patient list with next appointment, assigned provider, and which systems lack the value.
  2. Patient golden-record compare — Click a patient name → side-by-side AthenaNet (golden) vs Zenoti vs GHL vs WooCommerce for the incomplete fields, with “copy from source” affordances where safe.
  3. Provider filter — Jump from outline / provider heading → that provider’s action lists only (matches the sample Doc’s per-provider structure).
  4. Appointment Date Range — Primary list filter (From / To, plus Today · Next 7 / 14 / 30 day presets). Scopes 5 Journeys Practice summary counts and provider “Upcoming Appointment Issues” to visits in the selected window; follow-ups and copays can optionally respect the same range or stay unscoped.
  5. Urgency window — Click “Appts in next 7 days” → same issues sorted by appointment proximity; presets above are the same control, not a separate system.
  6. Issue → workflow — From an Upcoming Appointment Issue row → deep link into AthenaNet chart, Zenoti profile, or GHL contact (system where the gap must be fixed).
  7. Assignment & audit — Assign a gap to front-desk / admin, mark resolved, and keep a short history so the daily report doesn’t resurface cleared items.
  8. Cross-system conflict — When values disagree (e.g. phone in GHL ≠ AthenaNet), surface as Conflict rather than merely Missing, then drill to reconcile against the golden record.
