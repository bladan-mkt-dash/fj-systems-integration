Build Gantt (imagined)

Countdown timeline for the build checklist: Day −60 at the left (kickoff) → Day 0 at the right (Admin Check List launch — daily action report + record ledger). Spine is easier single-entry sync + ops reporting; calendar / Woo / portal booking are supporting. Moderate pace for information gathering & architecture; aggressive for AI-assisted implementation.

Assumptions (art, not contract):
  • ~½ FTE 5 Journeys decision-maker during discovery; contractor is a competent AI-assisted engineer (not a full team).
  • Sandbox API access available by ~Day −29 (checklist item 13); no major vendor API blockers.
  • Day 0 = go-live of the live, internal-only Admin Check List app (Phase D + UAT) — staff-only (IAP/RBAC), HIPAA-conscious GCP posture — not publishing the static HTML wireframe.
  • Build hygiene in scope: least-privilege roles, session timeout, Health Coach / Nutritionist calendar shows blocked time only by default (no patient details), access logging. Excludes shoulder-surfing / screen left open in public (workforce safeguard).
  • Phase E (cancels/declines) may land soft / follow-on; Phase F (patient-portal self-booking) stays deferred on Zoom / HIPAA–BAA.
  • Overlap is intentional where dependencies allow (e.g. design artifacts with late joint work).
  • Calendar days shown; weekends included as buffer rather than drawn separately.
