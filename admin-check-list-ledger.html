Admin Check List — Record Ledger

Provider-scoped completeness ledger: each patient appears once, with a collected vs missing view of chart / payment / membership fields drawn from AthenaNet & Zenoti.

Layout inspired by the Q3 admin workbook pattern · illustrative data only · fictitious names · not a live chart extract

Looking for the daily action / issues layout instead? See the original Admin Check List wireframe.

Design shift: The spreadsheet often lists the same patient under multiple appointment dates. This wireframe collapses to one row per patient under each provider, and turns the Yes / No / N/A columns into a ledger of what is collected vs still missing. Use Appointment Date Range to scope the ledger to patients whose next visit falls in the window (same control pattern as the daily action report).
